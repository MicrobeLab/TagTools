# TagTools

**TagTools** is a single-cell demultiplexing tool that does not require prior knowledge of sample genotypes. It features high computational efficiency and high detection rates for singlets and doublets. Currently, **TagTools** only supports Linux systems.

------------------------------------------------------------------------

## Prerequisites

-   Python 3.8 or higher (Anaconda distribution recommended) with the following packages:

    -   `numba`
    -   `numpy`
    -   `scikit-learn`
    -   `matplotlib`

-   External tools that should be available in shell \$PATH:

    -   `mawk`
    -   [`GLIMPSE_phase` (v1)](https://odelaneau.github.io/GLIMPSE/glimpse1/index.html)
    -   [`GLIMPSE_chunk` (v1)](https://odelaneau.github.io/GLIMPSE/glimpse1/index.html)
    -   [`demuxlet`](https://github.com/statgen/demuxlet)
    -   `bcftools` (v1.19 or higher)
    -   `samtools` (v1.19 or higher)
    -   `tabix`
    -   `bgzip`

------------------------------------------------------------------------

## Modules

TagTools consists of four modules:

1.  `make_ref.py`
2.  `cluster_predict.py`
3.  `imputed.py`
4.  `demultiplexing.py`

------------------------------------------------------------------------

## Example files

A demo dataset is available for download [here](https://figshare.com/s/1ea477d113abe5891e1e). It includes a simulated 3-sample mixture with 500 droplets. The sample information for each droplet is provided in `cb-with-sample-ID.txt`. The full list of cell barcodes is in `all-cb-list.txt`, and a subset for clustering testing is in `test-cb-list.txt`.

------------------------------------------------------------------------

## Usage

### Step 1: `make_ref.py`

Create a haplotype panel from VCF files.

```{bash}
python make_ref.py -h
usage: make_ref.py [-h] --vcf-list VCF_LIST --out-name OUT_NAME --nt NT
                   [--add-chr] [--rm-chr]
                   
Create haplotype panel from VCF files.

options:
  -h, --help           show this help message and exit
  --vcf-list VCF_LIST  Path to a text file containing a list of bgzip-compressed haplotype VCF files.
  --out-name OUT_NAME  Prefix for the output file names.
  --nt NT              Number of threads to use for processing.
  --add-chr            Whether to add the 'chr' prefix to the haplotype panel.
  --rm-chr             Whether to remove the 'chr' prefix from the haplotype panel.
```

To create a haplotype panel compatible with **TagTools**, begin by downloading the haplotype panel files in VCF format.

For the hg38 reference genome, the panel files can be obtained from [here](http://ftp.1000genomes.ebi.ac.uk/vol1/ftp/data_collections/1000G_2504_high_coverage/working/20220422_3202_phased_SNV_INDEL_SV/).For the hg19 reference genome, the panel files can be obtained from [here](https://ftp-trace.ncbi.nih.gov/1000genomes/ftp/release/20130502/).

Save the paths to all autosomal haplotype VCF files in a text file (e.g., `vcf_files.txt`), with one VCF file path per line. Ensure the files are in the default bgzip-compressed format as downloaded.

**Note:** The haplotype panel generated by `make_ref.py` only needs to be created once and can be reused for subsequent analyses. Recreate the panel only if the reference genome for single-cell alignment changes (e.g., when switching from hg38 to hg19).

**Example command using the demo dataset:**

```{bash}
python make_ref.py --vcf-list vcf_files.txt --out-name 1000-genome-38-for-imp-2 --rm-chr --nt 48
```

**Parameter details:**

-   `1000-genome-38-for-imp-2`: The output file prefix

-   `--rm-chr` indicates that the 'chr' prefix should be removed from all chromosome identifiers in the haplotype data (ensuring consistency with the chromosome naming convention used in the reference genome for single-cell sequencing reads alignment).

**Output files:**

-   `1000-genome-38-for-imp-2.bcf` (and .csi index): The haplotype panel file

-   `1000-genome-38-for-imp-2.site.gz` (and .csi index): SNP site file containing all variants from the haplotype panel

-   `1000-genome-38-for-imp-2.MAF0.05.site.gz` (and .csi index): SNP site file filtered for variants with Minor Allele Frequency (MAF) \> 0.05

-   `1000-genome-38-for-imp-2.region`: Chunked region file dividing the genome into intervals for parallel processing

### Step 2: `cluster_predict.py`

This module performs genotype similarity detection and clustering for single cells by taking as input: a cell barcode list, a single-cell sequencing BAM file, and a set of SNP sites for variant calling.

```{bash}
python cluster_predict.py -h    

usage: cluster_predict.py [-h] --bam BAM --out-name OUT_NAME --cb-list CB_LIST --snp-site SNP_SITE --ref REF --sample-num SAMPLE_NUM [--nt NT] [--plot-log]

Cluster prediction from SNP data

options:
  -h, --help            show this help message and exit
  --bam BAM             BAM file, should be sorted and index by samtools
  --out-name OUT_NAME   out file name
  --cb-list CB_LIST     cell barcode list file
  --snp-site SNP_SITE   common SNPs site file, indexed with tabix
  --ref REF             reference genome file
  --sample-num SAMPLE_NUM
                        number of sample multiplexed
  --nt NT, --threads NT
                        number of threads to used, default: 16
  --plot-log            plot the log file, default: false
```

**Example command using the demo dataset:**

```{bash}
python cluster_predict.py --cb-list test-cb-list.txt --bam keep-bc-sub.bam --snp-site 1000-genome-38-for-imp-2.MAF0.05.site.gz --sample-num 3 --nt 48 --plot-log --ref genome.fa --out-name clustering-test
```

**Parameter details:**

-   `test-cb-list.txt`: Cell barcode list file for testing

-   `keep-bc-sub.bam`: Mixed-sample BAM file

-   `1000-genome-38-for-imp-2.MAF0.05.site.gz`: Common SNP sites (MAF \> 0.05) from the 1000 Genomes Project

-   `--sample-num 3`: Number of multiplexed samples

-   `--ref genome.fa`: Reference genome file for genotype likelihood calculation

**Output files:**

-   `clustering-test.pl.gz`: Genotype likelihood values for each droplet (each line represents PL values for one SNP per sample, with the last three columns containing the PL values)

-   `clustering-test.pl.gz.csi`: Index file for the above

-   `clustering-test-predict.txt`: Demultiplexing predictions for droplets through spectral clustering

-   `clustering-test-predict.singlet`: Clustering group IDs for all singlets

-   `clustering-test.inertia.pdf`: Elbow curve plot showing the inflection point for different cluster numbers (k-values), used to determine the optimal k-value (generated only when using `--plot-log`)

-   `clustering-test-best-k.txt`: Raw data used for generating the elbow curve

### Step 3: `imputed.py`

This module performs low-depth sequencing imputation using sequencing data from different droplet groups to obtain reliable genotype calls for each group.

```{bash}
python imputed.py -h
usage: imputed.py [-h] --bam BAM --out-name OUT_NAME --ref REF --snp-site SNP_SITE --hap HAP --chunk CHUNK --cb-group CB_GROUP [--nt NT]

Imputed haplotype using groups of cell.

options:
  -h, --help            show this help message and exit
  --bam BAM             BAM file, should be sorted and index by samtools
  --out-name OUT_NAME   out file name
  --ref REF             reference genome file
  --snp-site SNP_SITE   SNPs site file of the haplotype panel, indexed with tabix
  --hap HAP             haplotype panel file
  --chunk CHUNK         chunked region file
  --cb-group CB_GROUP   cell barcode group file
  --nt NT, --threads NT
                        number of threads to used, default: 16
```

**Example command using the demo dataset:**

```{bash}
python imputed.py --bam keep-bc-sub.bam --out-name imputed-test --ref genome.fa --nt 48 --snp-site 1000-genome-38-for-imp-2.site.gz --hap 1000-genome-38-for-imp-2.bcf --chunk 1000-genome-38-for-imp-2.region --cb-group clustering-test-predict.singlet
```

**Parameter details:**

-   `1000-genome-38-for-imp-2.bcf`: Haplotype panel file created in Step 1

-   `1000-genome-38-for-imp-2.site.gz`: SNP site file containing all variants from the haplotype panel (including those beyond MAF \> 0.05)

-   `1000-genome-38-for-imp-2.region`: Chunked region file for genomic interval partitioning

-   `--cb-group clustering-test-predict.singlet`: Group ID file for singlets obtained from clustering in Step 2

**Output files:**

-   `imputed-test-imp-raw.bcf`: Imputed genotype data for each group

-   `imputed-test-imp-raw.bcf.csi`: Index file for the imputed BCF\

### Step 4: `demultiplexing.py`

This module performs sample demultiplexing using demuxlet, taking as input: a mixed-sample single-cell sequencing file, a cell barcode list file, and an imputed genotype file.

```{bash}
python demultiplexing.py -h
usage: demultiplexing.py [-h] --vcf VCF --bam BAM --cb-list CB_LIST --out-dir OUT_DIR [--nt NT] [--info INFO]

Demultiplexing using demuxlet.

options:
  -h, --help            show this help message and exit
  --vcf VCF             imputed VCF/BCF file, indexed with bcftools
  --bam BAM             BAM file, sorted and index with samtools
  --cb-list CB_LIST     cell barcode list file
  --out-dir OUT_DIR     out directory name
  --nt NT, --threads NT
                        number of threads to used, default: 16
  --info INFO           INFO score to filter imputed genotype, default: 0.4
```

**Example command using the demo dataset:**

```{bash}
python demultiplexing.py --bam keep-bc-sub.bam --cb-list all-cb-list.txt --out-dir demultiplexing-test --nt 48 --info 0.4 --vcf imputed-test-imp-raw.bcf
```

**Parameter details:**

-   `all-cb-list.txt`: Cell barcode list for demultiplexing

-   `demultiplexing-test`: Output directory name

-   `imputed-test-imp-raw.bcf`: Input file containing imputed genotypes

-   `--info 0.4`: Filter threshold retaining SNPs with INFO score \> 0.4 for demultiplexing

**Output files:**

-   `demuxlet-merge.vcf.gz`: Filtered genotype file containing SNPs with INFO \> 0.4 and MAC \> 1

-   `demuxlet.best`: Primary demultiplexing results file

-   `demuxlet.single`: Statistics generated by demuxlet for matching each cell with each possible sample

-   `demuxlet.sing2`: Statistics generated by demuxlet for matching each cell with each possible configuration of doublet
